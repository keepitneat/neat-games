## 🟡 Should-fix

1. **`shared/theme.js` — new DOM helpers ship with zero test coverage.** `applyTheme` + `wireThemeToggle` are now the single source of truth for three games, but `theme.test.js` only covers the pure helpers. Nothing asserts click→`data-theme`, `system`→`removeItem` (not `setItem('theme','system')`), or `aria-pressed` exclusivity. The whole point of NEAT-94 is one bug now breaks all three games — that deserves a test. `wireThemeToggle(doc)`'s injectable param already invites it; inject storage too and assert (a) click→attribute, (b) system→removeItem, (c) aria-pressed exclusivity, (d) init paint. The removeItem-vs-setItem assertion specifically guards the FOUC contract.

2. **`crowns/app.js:211` — `renderSettings()` no longer paints the theme buttons.** Previously repainted `aria-pressed` every render; now painted once at boot by `wireThemeToggle` (handler repaints on click). Fine today — but if any future "reset settings" path mutates `localStorage.theme` without going through the buttons, they'd show stale `aria-pressed`. Should-fix-as-awareness: a one-line note or conscious "intended."

3. **Migration-bookkeeping comments narrate *what*, not *why*.** The "now lives in /styles.css" banners across the three game CSS files restate the diff. Trim to one short line each (or delete). Keep the genuinely load-bearing one: minesweeper's note that `.overlay-sub` / `ms-pop` / larger `h2` intentionally stayed local (a real "why didn't this move?" question).

## Discussion

### The theme consolidation is correct — and that's why it needs the one test it lacks
The convention chain holds: `wireThemeToggle` writes `removeItem('theme')` for system / `setItem` otherwise, and all three FOUC inline scripts gate on `saved && saved !== 'system'` — "system = key absent" is consistent across both paths. The refactor silently FIXED a latent minesweeper bug: its old `wireTheme` did `setItem('theme', mode)` unconditionally, storing the literal `'system'` and breaking its own FOUC check. That wrong-then-right behavior is now load-bearing for all three and asserted by nothing — strongest argument for S-1.

### Background unification trades intra-games consistency for hub-vs-games inconsistency
Before: hub/crowns/2048 shared mint, minesweeper odd one out. After: three games uniform neutral, hub odd one out. Net better, and the crowns `.win` border/shadow is a thoughtful compensation — but "hub looks different from everything it links to" should be named before it ships (NG-1). Wants a human eye + mobile render.

### Comments are doing migration-bookkeeping, not why-explaining
Keep only minesweeper's "what intentionally stayed local" note; trim the rest — the comment-heaviness Ashlie trims by hand anyway.
